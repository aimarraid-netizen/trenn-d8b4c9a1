**Goal:** Ehitada ümber treeningute analüüsi süsteem SQLite-põhiseks, varustus-teadlikuks, mobile-first HTML-väljundiga ja reaalajas Kratt-integratsiooniga — kõrvaldades praegused võltsregressid ja andmevead.

**Architecture:** SQLite andmebaas (`trenn.db`) keskmes. Failid jõuavad otse Telegrami Kratile (Google Drive + rclone + n8n kaovad). Jõutrenn = Gymaholicu **üksik-trenni CSV** (üks trenn korraga). **Tech Stack:** Python 3, SQLite (stdlib `sqlite3`), fitparse (FIT), Chart.js (HTML graafikud), Jinja2 või f-string HTML. Git/GitHub Pages hostimiseks. (Üksik-CSV parsimine = stdlib, pandas pole vajalik.)

## Andmemudel (SQLite skeem)

```sql
-- Trennisessioon
CREATE TABLE workouts (
    id INTEGER PRIMARY KEY,
    timestamp TEXT NOT NULL,        -- "2026-05-21 15:55:00"
    date TEXT NOT NULL,             -- "2026-05-21"
    workout_name TEXT,
    workout_type TEXT,              -- selg/rind/jalad/strength/cardio
    duration_min INTEGER,
    total_volume REAL,
    avg_hr INTEGER,
    notes TEXT,
    source TEXT,                    -- gymaholic/fit/gpx/manual
    UNIQUE(timestamp, workout_name) -- dedup
);

-- Üksik seeria (mitte grupeeritud — grupeerimine toimub kuvamisel)
CREATE TABLE sets (
    id INTEGER PRIMARY KEY,
    workout_id INTEGER NOT NULL REFERENCES workouts(id) ON DELETE CASCADE,
    exercise_name TEXT NOT NULL,
    set_number INTEGER,             -- järjekord trennis
    reps INTEGER,
    weight_kg REAL,                 -- NULL kui kaalu pole logitud (EI 0.0!)
    equipment TEXT,                 -- trx/machine/barbell/dumbbell/bodyweight/cable — NULL=peri vaikest
    total_volume REAL,
    duration_sec REAL,
    max_hr INTEGER,
    avg_hr INTEGER,
    kcal INTEGER,
    note TEXT                       -- lives kontekst: "väsinud", "uus tehnika"
);

-- Harjutuse vaikeseaded (püsiv varustus + rep-vahemik strateegiast)
CREATE TABLE exercises (
    name TEXT PRIMARY KEY,
    default_equipment TEXT,         -- nt Face Pull -> "trx"
    target_sets INTEGER,
    target_reps_min INTEGER,
    target_reps_max INTEGER,        -- NULL kui "max" (Plank, Leg Raise)
    muscle_group TEXT               -- selg/rind/jalad/õlad/käed/kõht
);

-- Rekordid arvutatakse baasist päringuga, ei salvestata eraldi (üks tõeallikas)
```

**Kriitiline reegel:** `weight_kg = NULL` tähendab "kaalu pole logitud" (TRX/kehakaal/puuduv). EI tohi olla `0.0`, mida võrreldakse kui tugevuse langust. Progressi arvutus jätab NULL-kaalu harjutused kordusepõhisele võrdlusele.

## FAAS 2 — Parserid (sisend otse Telegrami)

### Task 2.1: `v2/parse_gymaholic_csv.py` — üksik-trenni CSV
Gymaholicu üksik-trenni eksport on **semikooloniga (`;`) eraldatud CSV**, mitte tabelvorm. Formaat:
```
;-----------------
;3. Selg & biitseps          <- trenni nimi
;-----------------
;Date;May 21., 15:55         <- kuupäev (AASTA PUUDUB -> tuleta jooksvast aastast)
;Duration;0h:58m            <- kestus
;KCAL;380                   <- trenni kokku
;Heart rate;113 bpm         <- trenni keskmine pulss

#;Bent Over Barbell Row;REPS;TIME;REST   <- harjutus + veerud
N;6-10 reps                 <- REP-VAHEMIKU EESMÄRK (inline! -> exercises tabel)
1;;70 kg x 6;;0:00          <- seeria: nr;;kaal x kordused;aeg;rest
2;;70 kg x 6;;0:00
3;;70 kg x 6;;0:00
```
Parsimisreeglid:
- Rida algab `#;` → uus harjutus (nimi = 2. väli)
- Rida algab `N;` → rep-vahemik (nt "6-10 reps" → target_reps_min=6, max=10; "max"/"Pulss..." → NULL)
- Rida algab numbriga → seeria. Väli 3 = `"70 kg x 6"` → parsi kaal+kordused regexiga. Tühi kaal (cardio) → NULL kaal, väli 4 = TIME
- **Kaal on JUBA kilodes** — EI jaga 100-ga (erinevus ZIP-ist!)
- Kuupäev: `"May 21., 15:55"` → lisa jooksev aasta → `datetime`. Kui kuu > praegune kuu, eelmine aasta (aastavahetuse käsitlus)
- **Per-seeria pulss/kalorid puuduvad** — jäta NULL (otsustatud: müra). Salvesta ainult trenni koond-pulss (113) workouts.avg_hr ja KCAL
- Võtab faili vastu Telegramist → loeb → kirjutab SQLite
- **Test:** `tests/test_parse_csv.py` — parsib näidisfaili (eilne "3. Selg & biitseps"), kontrollib: 6 harjutust, Bent Over Row 3×(70kg,6), rep-vahemik 6-10 salvestus, kuupäev 2026-05-21
